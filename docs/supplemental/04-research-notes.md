# 研究笔记

## 研究问题

四维图新命题关注「AI 助力数据治理」，真正值得突破的点在于让数据治理从后台工程走向业务现场。研究过程中，我们把问题拆成三个层次：

1. 企业有哪些数据已经存在，却没有真正转化为能力？
2. 业务侧为什么很难把自己的需求快速产品化？
3. Agent 如何在权限、安全、证据和组织记忆约束下落地？

## 关键洞察

### 洞察一：企业更需要可复用的判断结构

CRM 记录客户，项目系统记录进度，合同系统记录条款，会议纪要记录沟通。但真实业务判断往往发生在这些系统之间。例如「这个客户下次拜访该问什么」「这个项目为什么容易延期」「这个指标变化是不是异常」，答案需要跨系统理解上下文。Helios 的设计重点是把这些跨系统判断沉淀为 Agent 能力，持续超越单系统查询。

### 洞察二：业务侧最懂需求，却最难把需求变成系统能力

传统企业应用建设依赖研发排期。业务人员知道自己每天重复问什么、查什么、判断什么，但他们很难把经验变成可复用工具。Helios 把自然语言作为能力生产入口：业务人员描述目标、数据范围、判断规则和输出格式，系统将其转化为可执行 Agent。研发侧的角色从不断响应零散需求，转向维护底层连接器、权限模型、审计机制和可复用能力组件。

### 洞察三：权限应该跟着业务关系走

企业 Agent 最大风险之一是越权调用。普通聊天机器人容易把「能检索到」误当成「可以回答」。Helios 将权限放在业务侧治理：不同角色、客户、项目、合同和指标口径对应不同访问边界。Agent 在执行任务前先判断用户身份、业务关系和场景敏感度，再决定可访问的数据、可执行动作和是否需要人工确认。

### 洞察四：可信答案需要证据账本，而不只是更好的提示词

企业管理决策不能依赖一段看似流畅的总结。Helios 为关键结论绑定来源、版本、时间、口径、置信度和推理路径。这样一来，答案可以被复核，风险可以被追踪，管理层也能看到结论背后的依据。证据账本使 Agent 从「会说」走向「可承担业务责任」。

### 洞察五：组织记忆是 Agent 的长期护城河

通用模型可以提供基础推理能力，但真正的企业价值来自组织自己的项目经验、客户语境、业务规则和人工修正。Helios 会把高质量问答、负责人确认、错误修正、项目复盘沉淀回知识网络和 Agent 模板中。系统使用越多，越理解企业自己的做事方式。

## 产品原则

### 原则一：自然语言即能力入口

用户不需要写 SQL、流程图或接口文档，只需要说清楚业务目标。Helios 将自然语言转化为任务、数据调用、判断规则和输出模板。

### 原则二：能力可复用，答案可沉淀

每一次高质量问答都可以转成模板，每一次人工修正都可以变成规则，每一次复盘都可以成为下次任务的前置检查。

### 原则三：权限在业务侧，能力在平台侧

平台提供连接器、编排、审计和模型能力；业务侧定义角色、边界、流程和确认机制。这样既能放开创新，又能守住数据安全。

### 原则四：先做高频闭环，再扩展复杂场景

Helios 适合从客户洞察、项目复盘两个高频场景开始。它们数据源明确、痛点强、复用价值高，也容易验证成效。之后再扩展到合同风控、经营分析、售前支持和知识管理。

## 用户旅程草图

1. 业务人员提出自然语言需求：「我想做一个客户拜访准备 Agent。」
2. Helios 追问关键配置：「需要哪些客户字段？要不要读合同？输出给谁看？是否包含风险判断？」
3. 业务负责人确认权限和审批边界。
4. 系统生成 Agent 任务流和输出模板。
5. 用户在真实业务中调用 Agent。
6. 人工修正高风险判断。
7. 高质量结果回流为模板和知识规则。

## 风险与应对

| 风险 | 表现 | 应对 |
| --- | --- | --- |
| 数据质量不稳定 | 系统答案依据不足或口径冲突 | 引入来源标注、口径校验和置信度分级 |
| 权限越界 | Agent 调用不该访问的数据 | 业务角色、项目关系、敏感场景三层权限判断 |
| 业务人员不会配置 | Agent 创建门槛高 | 用自然语言问答式配置和模板推荐降低门槛 |
| 经验沉淀质量参差 | 低质量问答污染知识库 | 引入人工确认、复用次数和质量评分 |
| 试点场景太大 | 难以快速验证价值 | 从客户洞察和项目复盘两个场景启动 |

## 阶段性路线

### 第 1 阶段：可演示原型

完成展示页、核心叙事、Agent 模板样例和模拟数据分析，证明方案表达完整。

### 第 2 阶段：场景试点

选择一个销售场景和一个项目复盘场景，接入样例数据源，完成问答、证据链和模板沉淀闭环。

### 第 3 阶段：业务侧工作台

让业务负责人可以配置 Agent 的数据范围、执行动作、审批边界和输出格式。

### 第 4 阶段：能力资产库

沉淀跨部门 Agent 模板，形成客户洞察、项目复盘、合同风险、经营分析等能力资产。

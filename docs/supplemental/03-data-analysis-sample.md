# 数据分析样本

## 样本说明

以下为 Helios 在企业数据治理场景中的模拟分析样本，用于展示系统如何把分散业务数据转化为可执行的 Agent 能力。样本不包含真实客户数据，可作为试点时的数据结构、指标口径和输出样式参考。

## 场景 A：客户洞察 Agent

### 业务问题

销售准备拜访重点客户，希望快速了解该客户的历史合作、当前项目风险、合同节点、客户关注点和下一步沟通建议。

### 输入数据样本

| 数据源 | 字段 | 示例 | 用途 |
| --- | --- | --- | --- |
| CRM | 客户名称、行业、联系人、商机阶段 | A 客户、车企、采购负责人、方案评估中 | 建立客户基础画像 |
| 项目库 | 项目名称、周期、负责人、状态 | 高精地图数据服务、2024 Q2-Q4、交付中 | 判断历史合作与当前进展 |
| 合同摘要 | 合同金额、付款节点、交付义务、违约条款 | 首付款 30%、验收后 50%、延期责任约定 | 识别合同风险 |
| 会议纪要 | 客户关注点、待办事项、承诺时间 | 数据更新频率、接口稳定性、月底前答复 | 提取真实沟通语境 |
| 指标口径表 | SLA、交付完成率、验收标准 | 月度更新率、接口可用性、缺陷关闭率 | 保证结论口径一致 |
| 历史复盘 | 相似项目问题、解决方案、经验标签 | 需求变更频繁、接口联调延期、提前锁定验收标准 | 迁移组织经验 |

### Agent 分析路径

1. 意图路由识别为「客户洞察 + 项目风险 + 拜访建议」复合任务。
2. 业务知识网络关联客户、项目、合同、会议和相似案例。
3. Agent 编排引擎按最小权限读取必要上下文。
4. 证据账本记录每条结论的来源、版本、时间和置信度。
5. 输出客户拜访卡片，并将高质量结论沉淀为客户洞察模板。

### 输出样例

| 输出模块 | 示例内容 | 证据来源 |
| --- | --- | --- |
| 客户概况 | A 客户近三年合作 4 个项目，其中 2 个进入持续服务阶段 | CRM、项目库 |
| 当前风险 | 当前项目存在验收口径未完全锁定风险，风险等级为中 | 合同摘要、会议纪要 |
| 客户关注点 | 客户近期高频关注数据更新频率、接口稳定性和问题响应速度 | 近 3 次会议纪要 |
| 相似经验 | 过去同类项目延期多由接口联调和验收标准变更引起 | 历史复盘库 |
| 拜访建议 | 优先确认验收口径、接口联调排期、双方责任人和风险升级路径 | Agent 综合推理 |

## 场景 B：项目复盘 Agent

### 业务问题

交付负责人希望复盘过去两年相似项目，提炼延期原因、客户争议点、成功经验和下次立项前的风险检查清单。

### 输入数据样本

| 项目 | 延期天数 | 主要原因 | 客户争议点 | 最终处理 | 可沉淀经验 |
| --- | ---: | --- | --- | --- | --- |
| P-001 | 18 | 需求边界变化 | 验收标准不清 | 补充验收清单 | 立项前锁定验收口径 |
| P-002 | 7 | 接口联调排期冲突 | 数据格式不一致 | 双方技术例会 | 早期建立接口责任人机制 |
| P-003 | 0 | 无 | 无 | 按期验收 | 关键节点周报机制有效 |
| P-004 | 25 | 客户侧审批延迟 | 交付时间争议 | 升级项目委员会 | 高风险客户需要提前设置升级通道 |
| P-005 | 12 | 数据质量返工 | 样本覆盖不足 | 扩充测试集 | 交付前增加样本覆盖检查 |

### 初步统计

| 指标 | 模拟结果 | 业务解释 |
| --- | ---: | --- |
| 平均延期天数 | 12.4 天 | 项目交付仍受需求变更和外部协同影响 |
| 主要风险集中度 | 60% 来自验收口径和接口联调 | 风险具备可识别模式，可被模板提前预警 |
| 可复用经验条目 | 5 条 | 每次复盘都可以沉淀为下一次立项前检查项 |
| 建议 Agent 模板 | 3 个 | 立项风险检查、验收口径检查、接口联调检查 |

### Helios 生成的复盘结论

过去两年相似项目的主要风险集中在三个环节：验收口径未提前固化、接口联调责任人不清、客户侧审批链路不透明。建议将项目启动前检查清单沉淀为「项目立项风险 Agent」，在立项阶段自动追问验收标准、接口责任人、客户审批路径、数据样本覆盖和风险升级机制。该 Agent 可以在每次项目复盘后自动更新检查项权重，使组织经验从事后总结转变为事前预防。

## 试点指标设计

| 指标 | 试点前基线 | 试点目标 | 采集方式 |
| --- | ---: | ---: | --- |
| 资料检索平均耗时 | 20 分钟 | 5 分钟以内 | 用户任务日志 |
| 跨部门重复问询次数 | 每周 100 次 | 下降 50% | IM/工单统计 |
| 关键结论证据链覆盖率 | 30% | 90% | 系统自动记录 |
| 复盘经验模板化数量 | 每月 0-3 个 | 每月 20 个以上 | Agent 模板库 |
| 新人完成业务背景理解耗时 | 3 天 | 1 天以内 | 新人任务评估 |

## 分析结论

Helios 的价值不只来自一次查询节省了多少时间，更来自长期沉淀的能力资产。客户洞察、项目复盘和合同风险这些场景都有共同特点：数据源分散、判断依赖经验、结论需要证据、流程可以复用。只要把这些场景转化为 Agent 模板，企业就能把原本分散在个人脑中的业务判断沉淀到系统中，让经验能够被下一次任务继续调用。

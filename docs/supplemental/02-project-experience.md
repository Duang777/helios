# 相关项目经验

## 项目名称

Helios：企业级业务能力沉淀 Agent 原型

## 项目背景

本项目基于开源工程进行二次改造，完成了 Helios 的品牌化、展示页、核心叙事和部署链路搭建。项目目标是验证一种更适合企业场景的 AI 产品形态：让业务人员通过自然语言把需求转化为可复用 Agent，并让每一次使用都沉淀为组织能力。

## 已完成内容

- 完成 Helios 项目命名、Logo 接入、首页视觉与 README 展示。

- 设计 Helios 六层架构：意图路由、业务知识网络、Agent 编排、权限治理、证据账本、组织记忆回流。
- 形成面向四维图新命题的产品叙事：业务沉淀成能力，自然语言交互即能力入口，权限在业务侧，Agent 在使用中持续进化。

## 项目能力体现

### 1. 产品抽象能力

我们将命题进一步抽象为「如何让业务经验沉淀成可运营的智能能力」。这个抽象能覆盖客户洞察、项目复盘、经营分析、合同风险、知识管理等多个场景，具备从单点工具升级为企业能力层的空间。

### 2. 架构设计能力

Helios 将 RAG、知识图谱、Agent 工作流、权限治理和证据追溯组合为一个闭环。每个模块都有明确边界：业务知识网络负责理解语义关系，Agent 编排引擎负责任务执行，证据账本负责可信输出，权限治理负责安全边界，组织记忆回流负责长期进化。

### 3. 前端表达与交付能力

项目已经完成在线展示页部署，能够以清晰视觉方式表达产品定位、核心价值和使用场景。展示页围绕「业务问题如何被转化为 Agent 能力」组织信息，让评委能快速理解方案落点。

### 4. 工程落地意识

方案保留了企业级系统落地时必须处理的关键问题：权限边界、数据来源、版本口径、人工确认、流程复用和组织记忆沉淀。它不依赖一次性大而全建设，可以从两个高频场景启动，再逐步扩展到更多业务线。

## 代表性场景

### 客户洞察 Agent

业务人员输入：「帮我整理 A 客户近三年合作情况、当前项目风险、关键联系人变化，以及下次拜访建议。」

Helios 拆解任务：

- 从 CRM 读取客户基础信息和联系人变化。
- 从项目库读取历史合作项目和当前进展。
- 从合同摘要读取交付义务、付款节点和风险条款。
- 从会议纪要提取客户关注点和未解决事项。
- 从相似项目中提取复盘经验。
- 输出客户画像、风险提示、建议问题清单，并附带证据链。

### 项目复盘 Agent

业务人员输入：「找出过去两年和车联网数据交付相关的项目，整理延期原因、客户争议点和可复用交付经验。」

Helios 拆解任务：

- 按项目类型、时间和关键词召回项目资料。
- 对会议纪要、交付记录、问题单进行结构化摘要。
- 归类延期原因、资源瓶颈、客户变更和验收风险。
- 生成复盘卡片，并沉淀为下次立项前的风险检查模板。

## 后续可扩展方向

- Agent 模板市场：沉淀销售、交付、法务、经营分析等部门可复用模板。
- 业务侧权限工作台：让业务负责人管理 Agent 可访问的数据范围和审批边界。
- 组织记忆评分：对问答质量、复用频率、人工修正次数进行量化，识别真正有价值的业务经验。
- 多模态资料接入：支持地图截图、项目图纸、合同扫描件、会议录音等非结构化资料。
